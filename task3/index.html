Audit and improve onboarding: sign flow

Funnel: task-specific landing → editor → step-by-step popovers → download → paywall. I audited the signing journey end-to-end and redesigned the in-product onboarding where activation leaks hardest.

Why sign flow

I tested two intent flows on pdf.net: sign and annotate. Both share the same structural problem: tours teach UI, not job completion. Sign has sharper conversion damage:

  • Tour step 3 points to a disabled "Place in PDF" button
  • Copy conflict: popover says "initials", field says "full name"
  • Full-screen modal hides the document users came to sign
  • Direct continuation of Task 1 signing-intent landing

Task 2 fixes paywall → purchase. This task fixes the leak before download: visitors who never place a signature never reach peak-intent paywall.

Sign landing → Upload → Editor → Signature modal + tour → Place signature → Download → Paywall

Current onboarding: what hurts conversion

Core issue: tour completion ≠ task completion. Users can click through all 3 steps without creating or placing a signature. The tour explains the product; it does not guide users to value.

Step 1/3: name field
Popover: "Type your initials". Placeholder: "Enter full name". Next works without typing.
Step 2/3: Type / Draw / Upload
Teaches 3 signature methods + 12 fonts before first value. Choice paralysis on a time-sensitive task.
Step 3/3: Place in PDF
Popover tells user to click CTA that is disabled until name is entered. Dead end.
After placement
"All done? Download" fires while signature is still selected. User may not feel finished; competes with repositioning.
Editor chrome
14 tools visible (Merge, Ask AI, etc.) for a user who came to sign one file. Sign tool not auto-selected from sign landing.

3

Action-gated coach marks (not a linear tour)

Replace the 3-step popover slideshow with max 2 contextual hints, each gated on user action:

Hint 1: panel open, field empty
"Enter your name as it should appear on the document." Show once. No Next button; dismiss on input.
Hint 2: name entered, not placed
"Click on the document to place your signature." Spotlight on canvas. Advance only after placement.
Hint 3: placed + idle
Download spotlight after signature placed AND (8–12s idle OR deselect). Copy: "Download your signed file · contract.pdf"

Remove entirely:

  • Tour step about Type / Draw / Upload tabs
  • "Skip tour" on a 2-hint flow
  • "All done?" before user finishes positioning
Cross-cutting pattern (annotate flow): The annotate tour ends on Undo (step 2/2) with no post-action nudge. Same fix applies: after first annotation stroke, idle-triggered Download spotlight. Sign flow is the primary case; completion nudge is a shared onboarding primitive.

Design principles

  • Job, not UI: onboarding steps map to sign → place → download, not modal tabs
  • Gated progression: next hint only after previous action completed
  • Context preserved: document visible during signature setup
  • One happy path: default font + type; alternatives available but not taught
  • Completion at peak intent: download nudge after value, before paywall

How to measure success

Primary metric: sign_tool_opened → signature_placed
